## 1. СХЕМА ПОТОКУ ДАНИХ

```
БАНКІВСЬКА ВИПИСКА (XLS/CSV/PDF)
        │
        ▼
  [БАНК / ІМПОРТ]
  автокласифікація по ЄДРПОУ, історії, ключовим словам
        │
        ▼
  bank_transactions ──────────────► contractors
        │                           (автоматичне створення)
        ▼
  [РЕЄСТР]
  класифікація + завантаження документів + AI розпізнавання
        │
        ▼
  transaction_items ──► stock_movements ──► [СКЛАД]
        │                                     │
        ▼                                     ▼
  [ВАЛІДАЦІЯ]                           [ЗБІРКИ]
  перевірка ПДВ                    компоненти → виріб
  is_validated = true
        │
        ▼
  [ЗВІТНІСТЬ]
  P&L, грошовий потік, заборгованість
  (тільки валідовані дані!)

  [КАСА] ─── паралельний потік готівки
  [КОНТРАГЕНТИ] ─── CRM + генерація документів
```

## 2. ЩОДЕННИЙ РОБОЧИЙ ПРОЦЕС

### Крок 1: Імпорт банківської виписки
**Модуль:** Банк / Звірка → вкладка "Імпорт"

1. Завантажте файл виписки (XLS, XLSX, CSV або PDF)
2. Система автоматично визначить формат (Monobank, ПУМБ, або AI для інших)
3. Перевірте автокласифікацію — напрям, стаття, контрагент
4. Дублікати позначені автоматично (зняті з вибору)
5. Натисніть "Імпортувати"

> **Автокласифікація працює по 5 правилах:**
> 1. ЄДРПОУ → контрагент з дефолтним напрямом/статтею
> 2. Історія операцій по ЄДРПОУ (найчастіша пара)
> 3. Історія по назві контрагента
> 4. Тип контрагента (клієнт/постачальник)
> 5. Ключові слова в описі ("зарплат" → Зарплата тощо)

### Крок 2: Класифікація в Реєстрі
**Модуль:** Реєстр

1. Відфільтруйте "Без статті" — це транзакції що потребують уваги
2. Для кожної: призначте напрям, статтю, проєкт
3. Завантажте первинний документ (PDF/фото) — AI розпізнає позиції
4. Перевірте розпізнані позиції — назви, кількості, ціни

### Крок 3: Валідація ПДВ
**Модуль:** Валідація

1. Відкрийте транзакцію — зліва дані, справа документ
2. Перевірте суми: "Разом позицій" vs "Банк"
3. Оберіть: **"Ціни БЕЗ ПДВ"** або **"Ціни З ПДВ"**
4. Перевірте перерахунок: Без ПДВ, ПДВ, Сума з ПДВ, Банк
5. Натисніть "Валідувати"

> **Система попереджає якщо:**
> - ПДВ > 20% (неправильний вибір типу цін)
> - Сума позицій не збігається з банком
> - Сума позицій > банк (від'ємний ПДВ)

**Масова валідація:**
- Пошук по контрагенту → виберіть чекбоксами → "Без ПДВ — валідувати"
- Для податків, зарплат тощо (без документів) — масово "Без ПДВ"

### Крок 4: Готівкові операції (за потреби)
**Модуль:** Каса

| Тип | Коли використовувати |
|-----|---------------------|
| Готівкова виручка | Отримано готівку від клієнта |
| Витрата готівкою | Оплата готівкою за товари/послуги |
| Видача підзвітних | Видача грошей працівнику |
| Повернення підзвітних | Працівник повертає залишок |
| Банк → Каса | Зняття готівки з рахунку |
| Каса → Банк | Внесення готівки на рахунок |

## 3. ЩОМІСЯЧНИЙ ПРОЦЕС

1. **Перевірити повноту імпорту** — всі банківські дні покриті виписками
2. **Класифікувати все** — Реєстр → фільтр "Без статті" → має бути 0
3. **Валідувати все** — Валідація → прогрес має бути 100%
4. **Інвентаризація** — Склад → "Інвентаризація" → ввести фактичні залишки
5. **Закрити підзвітні** — Каса → вкладка "Підзвітні" → прострочені = червоним
6. **Перевірити борги** — Звітність → "Заборгованість" → aging 90+ потребує уваги
7. **Звіт P&L** — Звітність → P&L за місяць, порівняти з попереднім

## 4. ТИПОВІ СЦЕНАРІЇ

### Отримали оплату від клієнта
1. Імпорт виписки → авто-класифікація "Доходи"
2. Реєстр → призначити статтю + проєкт
3. Завантажити рахунок/накладну → AI заповнить позиції
4. Валідація → "Ціни з ПДВ" → валідувати
5. Товари автоматично списуються зі складу (OUT)

### Закупили товар у постачальника
1. Імпорт виписки → авто-класифікація "Витрати"
2. Контрагенти → постачальник → "Створити документ" → "Прихідна накладна"
3. Завантажити файл накладної → AI заповнить позиції
4. Товари автоматично оприбутковуються на склад (IN)
5. Валідація → "Ціни без ПДВ" (зазвичай) → валідувати

### Виставити рахунок клієнту
1. Контрагенти → обрати клієнта → "Створити документ"
2. Тип: "Рахунок на оплату"
3. Додати позиції зі складу (пошук) або вручну
4. "Зберегти + PDF" → скачати та надіслати клієнту

### Створити видаткову накладну
1. Контрагенти → обрати клієнта → "Створити документ"
2. Тип: "Видаткова накладна"
3. Додати позиції → вони спишуться зі складу
4. "Зберегти + PDF"

### Збірка виробу
1. Збірки → "Нова збірка"
2. Назва виробу, кількість
3. Додати компоненти зі складу
4. "Провести збірку" → компоненти списуються, виріб оприбутковується

### Видача підзвітних
1. Каса → "Видача підзвітних"
2. Вказати кому, суму, термін
3. При поверненні: "Повернення підзвітних" → вказати суму повернення

## 5. ВАЖЛИВІ ПРАВИЛА

### Валідація — ключ до точних звітів
- **Невалідовані транзакції НЕ потрапляють у звіти** (P&L, Analytics, контрагенти)
- Валідуйте регулярно — інакше звіти будуть неповними
- Масова валідація прискорює роботу для однотипних операцій

### Склад — цілісність даних
- При видаленні позиції у Валідації — складський рух теж видаляється
- При додаванні нової позиції з товаром — рух створюється автоматично
- `computed_stock` = VIEW в PostgreSQL, оновлюється автоматично
- FIFO собівартість розраховується при кожному списанні

### Кредиторка/Дебіторка
- Базується на **неоплачених документах** (generated_docs без bank_transaction_id)
- Не на загальному балансі витрат/доходів!
- Прив'язка оплати до документа → борг зникає

### Прихідні документи
- Прихідна накладна = документ контрагента → зберігаємо оригінал, не генеруємо свій PDF
- При завантаженні файлу для AI — файл автоматично прикріплюється

| Звітність | KPI, P&L, грошовий потік, борги (aging 30/60/90+) | `/#/analytics` |

| Склад | Товари, залишки, рухи, інвентаризація | `/#/inventory` |
