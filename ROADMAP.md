# AiM Skill — Дорожня карта розвитку

## Поточний стан (v1.0)
- Управлінський облік (bank_transactions, P&L, Cash Flow)
- Складський облік (FIFO, збірки, рухи)
- Генерація документів (рахунок, накладна, акт)
- Контрагенти (CRM, контакти, договори, дебіторка/кредиторка)
- AI розпізнавання документів та реквізитів
- Звітність (4 вкладки: Огляд, P&L, Cash Flow, Заборгованість)

---

## Фаза 1 — Стабілізація та UX (1-2 тижні)

### 1.1 Мобільна версія
- Адаптивний дизайн для планшетів і телефонів
- Сторінка документів — мобільний сканер (камера → AI)
- Швидкий перегляд KPI на телефоні

### 1.2 Сповіщення та нагадування
- Прострочені дебіторки — сповіщення на дашборді
- Товари з низьким залишком — alert
- Документи в статусі "Чернетка" більше 7 днів
- Контрагенти без ЄДРПОУ

### 1.3 Пошук по всій системі
- Глобальний пошук (Cmd+K): товари, контрагенти, транзакції, документи
- Швидкий перехід до будь-якої сутності

---

## Фаза 2 — Автоматизація (2-4 тижні)

### 2.1 Автоматичне рознесення банківської виписки
- AI аналізує опис платежу → пропонує статтю, контрагента, проєкт
- Правила автоматичного рознесення (якщо ЄДРПОУ = X → стаття Y)
- Масове рознесення одним кліком

### 2.2 Автоматична звірка документів
- Рахунок → оплата (по сумі + контрагенту)
- Видаткова → оплата
- Прихідна → оплата постачальнику
- Статус документа автоматично змінюється при привʼязці

### 2.3 Шаблони операцій
- Регулярні платежі (оренда, зарплата, податки) — автостворення
- Шаблони для типових контрагентів

### 2.4 Email інтеграція
- Відправка документів (рахунок, акт) на email контрагента прямо з системи
- Статус "Відправлено" автоматично

---

## Фаза 3 — Розширена аналітика (3-5 тижнів)

### 3.1 Маржинальність по товарах
- Собівартість (FIFO) vs ціна продажу
- Маржа по кожному товару
- Топ прибуткових / збиткових товарів

### 3.2 Маржинальність по контрагентах
- Виручка vs собівартість проданих товарів
- Прибутковість кожного клієнта
- LTV контрагента

### 3.3 Аналітика по проєктах
- Бюджет vs факт по проєкту
- P&L по проєкту
- Рентабельність проєкту

### 3.4 Прогнозування
- Прогноз Cash Flow на основі планів + тренду
- Прогноз дебіторки (коли очікувати оплату)
- Сезонний аналіз (порівняння місяців YoY)

---

## Фаза 4 — Інтеграції (4-6 тижнів)

### 4.1 Банківська інтеграція
- Monobank API — автоімпорт виписок
- PrivatBank API
- Автоматичне оновлення щоденно

### 4.2 Податкова звітність
- Генерація податкових накладних
- Реєстр ПН для ЄРПН
- Декларація з ПДВ (допоміжний розрахунок)

### 4.3 1С / BAS інтеграція
- Експорт/імпорт даних
- Синхронізація довідників

### 4.4 Telegram бот
- Швидкий перегляд KPI
- Сповіщення про оплати
- Фото документа → AI розпізнавання → додати в систему

---

## Фаза 5 — Масштабування (6+ тижнів)

### 5.1 Мультикомпанія
- Кілька юридичних осіб в одній системі
- Консолідована звітність
- Переключення між компаніями

### 5.2 Права доступу
- Розширені ролі (бухгалтер бачить фінанси, менеджер — тільки свої проєкти)
- Аудит лог (хто що змінив)

### 5.3 Документообіг
- Погодження документів (workflow)
- Електронний підпис (КЕП інтеграція)
- Статуси: На погодженні → Погоджено → Підписано

### 5.4 API для зовнішніх систем
- REST API для інтеграції з іншими системами
- Webhooks для подій (новий документ, оплата)
