# Запуск Telegram-бота AiM Skill

Бот працює як **вебхук у тому самому Vercel-проєкті** (`api/telegram.js`) — окремий хостинг не потрібен, працює 24/7.

## Що вміє (за завданням)
- ➕ **Нова заявка** — майстер: компанія → контактна особа → телефон/email → потреба → джерело (рекомендація/тендер/холодний/інше). Заявка зʼявляється в системі зі статусом «Новий».
- 📋 **Заявки** — список усіх (з пагінацією), клік → повна картка + кнопка «Відкрити в системі».
- 🔍 **Пошук** — за номером заявки або назвою клієнта.
- 🔔 **Сповіщення власнику** — миттєво при створенні заявки (з усіма деталями).
- 🔐 **Доступ** — лише дозволені Telegram ID.

---

## Налаштування (кроки)

### 1. Створити бота в Telegram
1. Напиши **@BotFather** → `/newbot` → назва й username (напр. `aimskill_bot`).
2. Отримаєш **токен** (рядок виду `123456:ABC...`).

### 2. Дізнатись Telegram ID
- Кожному користувачу: написати боту `/start` (поки немає доступу — бот покаже його ID), або через **@userinfobot**.
- Запиши ID партнера, власника та ін.

### 3. Виконати міграцію 020 (Supabase → SQL Editor)
```sql
alter table orders add column if not exists lead_source text;
create table if not exists bot_sessions (
  telegram_id bigint primary key,
  state jsonb,
  updated_at timestamptz default now()
);
alter table bot_sessions enable row level security;
```

### 4. Додати змінні в Vercel (Settings → Environment Variables)
| Змінна | Значення |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен від BotFather |
| `TELEGRAM_ALLOWED_IDS` | дозволені ID через кому, напр. `11111111,22222222` |
| `TELEGRAM_OWNER_ID` | ID власника (кому слати сповіщення) |
| `APP_URL` | `https://aim-skill.vercel.app` (необовʼязково) |
| `BOT_API_TOKEN` | секрет для `POST /api/bot` (якщо користуєшся ним окремо) |

> `SUPABASE_SERVICE_KEY` уже має бути в проєкті (використовується поштою). Бот бере службовий доступ звідти.

Після додавання змінних — **Redeploy** у Vercel.

### 5. Підключити вебхук (один раз)
Відкрий у браузері (підстав свій токен):
```
https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=https://aim-skill.vercel.app/api/telegram
```
Має повернути `{"ok":true,...}`. Перевірити стан:
```
https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo
```

### 6. Готово
Напиши боту `/start`. Партнер може створити заявку за хвилину; власник одразу отримає сповіщення.

---

## Керування доступом
- Додати/прибрати користувача = змінити `TELEGRAM_ALLOWED_IDS` у Vercel і Redeploy.
- (Майбутнє) можна перенести список у таблицю, щоб власник керував без деплою.

## Усунення проблем
- **Бот не відповідає:** перевір `getWebhookInfo` (має бути правильний URL, без помилок), і що `TELEGRAM_BOT_TOKEN` заданий.
- **«Немає доступу»:** додай свій ID у `TELEGRAM_ALLOWED_IDS`.
- **Сповіщення не приходять власнику:** перевір `TELEGRAM_OWNER_ID` і що власник хоч раз написав боту `/start` (Telegram не дає слати першим, поки користувач не почав діалог).
